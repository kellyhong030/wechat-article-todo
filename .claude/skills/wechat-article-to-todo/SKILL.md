name: wechat-article-to-todo
description: 公众号文章转可执行 todolist。读取文章链接，自动提炼任务步骤，过滤广告，总结观点，按优先级排序，用户确认后执行。安全审查优先。
---

将公众号文章转化为可执行的 todolist 工作流。适合处理 AI 教程、工具推荐、使用指南类文章。

## 核心原则

- **内容为王，广告靠边** — 提取可执行内容，过滤营销/广告/付费推广
- **先确认，后执行** — 所有操作必须用户确认后才执行
- **安全第一** — 安装任何东西前先安全审查
- **要事优先** — 按重要程度和先后顺序排序 todolist
- **零遗漏** — 审计环节确保无遗漏

---

## 完整工作流程

### 第 1 步：读取文章

**输入方式**：用户提供公众号文章链接

**自动获取**（使用 imagenie.us 服务）：

```python
# API 调用
POST https://wechat.imagenie.us/extract
Content-Type: application/json

{
  "url": "https://mp.weixin.qq.com/s/XXXXX"
}

# 响应：直接返回 markdown 文本
```

**命令行工具**：
```bash
python3 tools/extract_from_imagenie.py "文章链接"
```

**其他输入方式（备选）**：
- 直接粘贴文章内容
- 提供本地文件路径

---

### 第 2 步：过滤广告

**必须过滤的内容**（不要纳入 todolist）：

| 类型 | 关键词/模式 | 处理方式 |
|------|------------|---------|
| 训练营/课程 | "训练营"、"付费课程"、"报名" | 直接忽略 |
| 知识星球/社群 | "知识星球"、"星球"、"社群" | 直接忽略 |
| 付费推广 | "扫码"、"付费"、"购买" | 直接忽略 |
| 二维码/推广链接 | `<img.*二维码.*>`、"关注公众号" | 直接忽略 |
| 营销话术 | "限时优惠"、"名额有限" | 直接忽略 |
| 课程推广 | "线下实战课"、"工作坊" | 直接忽略 |

**规则**：
- 遇到上述内容，不进入 todolist，不提示用户
- 如果整段都是广告，整段跳过

---

### 第 3 步：总结核心观点

**输出格式**：

```markdown
## 核心观点

1. **[观点 1]**：[简述]
2. **[观点 2]**：[简述]
3. **[观点 3]**：[简述]
```

**目的**：
- 快速理解文章核心信息
- 为后续提炼任务提供上下文
- 帮助用户判断文章价值

---

### 第 4 步：我的思考

**输出格式**：

```markdown
## 我的思考

这篇文章的核心启示：
- [启示 1]
- [启示 2]

对读者的价值：
- [价值 1]
- [价值 2]
```

**目的**：
- 展示 AI 对文章的深度理解
- 提供独特视角
- 为任务提炼提供思路

---

### 第 5 步：提炼可执行任务

**任务类型识别**：

| 任务类型 | 识别特征 | todolist 格式 |
|---------|---------|--------------|
| 工具安装 | "安装"、"brew"、"npm"、"pip" | `[安装] 工具名称: 安装目的` |
| 配置修改 | "配置"、"config"、"设置" | `[配置] 文件路径: 配置内容` |
| 代码编写 | "代码"、"function"、"class" | `[编码] 文件名: 功能描述` |
| 文件操作 | "创建文件"、"删除"、"移动" | `[文件] 文件路径: 操作说明` |
| 学习理解 | "学习"、"理解"、"掌握" | `[学习] 主题: 关键信息` |
| Agent 创建 | "Agent"、"协作"、"团队" | `[架构] 描述: 说明` |

**优先级排序规则**（要事优先）：

1. **最高优先级**：基础环境/工具安装（依赖项）
2. **高优先级**：核心功能代码/配置
3. **中优先级**：优化/改进/扩展功能
4. **低优先级**：知识点/备注

**输出格式**：

```markdown
## 可执行任务

### 优先级 1（最高）
- [ ] **[类型]** 任务描述

### 优先级 2（高）
- [ ] **[类型]** 任务描述

### 优先级 3（中）
- [ ] **[类型]** 任务描述
```

**文件组织**：
- 所有任务相关文件将创建在独立任务文件夹：`tasks/task-YYYY-MM-DD-XXX/`
- 除非用户明确指定其他位置

---

### 第 6 步：Review 任务遗漏

**检查清单**：

**准确性检查**：
- [ ] 任务描述是否清晰？
- [ ] 优先级是否合理？
- [ ] 任务类型是否正确？

**完整性检查**：
- [ ] 是否遗漏了文章提到的关键步骤？
- [ ] 是否遗漏了重要的依赖项？
- [ ] 文章中的示例代码是否都已提取？
- [ ] 是否误过滤了非广告内容？

**输出格式**：

```markdown
## 任务审计报告

- [x] 准确性：通过
- [x] 完整性：通过（已检查 N 个任务，无遗漏）
- [ ] 安全性：需进一步确认（如果有安装/修改操作）
```

---

### 第 7 步：确认任务、优先级、边界、权限

**向用户展示**：

```markdown
## 待确认执行清单

### 任务概览
- 总任务数：N 个
- 优先级分布：高 N 个、中 N 个、低 N 个

### 详细任务列表

#### 优先级 1（最高）
- [ ] **[安装]** 工具A
  - **目的**：[说明]
  - **边界**：[会做什么，不会做什么]
  - **权限**：[需要的权限]

#### 优先级 2（高）
- [ ] **[配置]** 文件B
  - **目的**：[说明]
  - **边界**：[会修改什么，不会修改什么]
  - **权限**：[需要的权限]

### 执行边界
会执行：[列举会执行的操作]
不会执行：[列举不会执行的操作]

### 风险评估
需要特别注意：[如有风险，说明]

---

**请确认**：
1. 以上任务是否准确？
2. 是否有遗漏或需要调整？
3. 是否开始执行？

回复"确认"开始执行，或提出修改意见。
```

---

### 第 8 步：安装前检查与执行

**如果是安装 skill/mcp/agent**：

#### 8.1 核实是否已存在

```bash
# 检查 skill 是否已存在
ls ~/.claude/skills/ | grep "[skill-name]"

# 检查 agent 是否已存在
ls ~/.claude/agents/ | grep "[agent-name]"

# 检查 mcp 是否已配置
cat ~/.claude/mcp.json | grep "[name]"
```

#### 8.2 符合铁律检查

**工具安装铁律**：

1. **优先装官方的**：
   - `anthropics/skills`
   - `obra/superpowers`
   - Star 多不等于安全，但官方出品有人背书

2. **安装前审查**：
   - 打开 `SKILL.md`/`README.md` 检查
   - 查找：Base64 编码、外部 URL、可疑的 curl/wget 命令

3. **沙箱测试**（如适用）：
   - 含脚本的 skill，先在 Docker 容器跑一遍
   - 命令：`docker run --rm -v $(pwd):/app alpine:latest sh -c "脚本内容"`

4. **危险操作二次确认**：
   - 涉及：发邮件、转账、删文件
   - 必须问："这将执行 [危险操作]，确认继续？"

5. **定期更新**：
   - 更新到最新版
   - 安全补丁及时打

#### 8.3 使用 skill-vetter 审查（如有）

```bash
# 使用 skill-vetter 审查 skill
skill-vetter review [skill-path]
```

#### 8.4 执行安装

**通过所有检查后，才执行安装**：

```bash
# 安装 skill
claude skill install [skill-path]

# 安装 agent
cp -r [agent-path] ~/.claude/agents/

# 安装 mcp
# （根据具体 mcp 的说明操作）
```

## 沟通规范

### 必须问用户的

| 时机 | 问什么 |
|------|--------|
| 第 7 步 | 任务是否准确？是否开始执行？ |
| 第 8 步 | **先进行安全审查，带着审查结果来决策是否安装** |

**重要规则**：安装任何工具前，必须先完成安全审查，并将审查结果展示给用户，由用户决策是否安装。不得未审查就直接安装。

### 不需要问用户的

| 事项 | 直接做 |
|------|--------|
| 文章提取 | 自动调用 imagenie.us API |
| 广告过滤 | 按规则自动过滤 |
| todolist 排序 | 按优先级规则自动排序 |
| 审计检查 | 自动执行检查清单 |

---

## AI 绝不应该做的

- 用户未确认就开始执行
- 不展示审计报告就执行
- 安装非官方来源的工具不提醒用户
- 执行脚本前不二次确认
- 将广告/营销内容纳入 todolist
- 跳过审计环节
- 不说明执行边界
- 不检查就安装 skill/mcp
- 不符合铁律就安装
- **未完成安全审查就请求用户确认安装**

## 文件组织规则

### 任务文件隔离

**默认行为**：每次任务创建独立文件夹，保持 skill 目录干净

```
项目根目录/
├── .claude/skills/wechat-article-to-todo/    # skill 本身（保持干净）
├── tasks/                                   # 任务输出目录
│   ├── task-YYYY-MM-DD-001/              # 按日期和序号命名
│   │   ├── article.md                  # 提取的文章原文
│   │   ├── todolist.md                 # 生成的 todolist
│   │   ├── summary.md                  # 核心观点总结
│   │   ├── thinking.md                 # AI 思考和分析
│   │   └── review.md                  # 任务审计报告
│   ├── task-YYYY-MM-DD-002/
│   └── ...
└── README.md
```

### 文件命名规范

**任务文件夹**：`task-YYYY-MM-DD-序号`
- 例如：`task-2026-03-18-001`
- 例如：`task-2026-03-18-002`

**任务文件**：
- `article.md` - 提取的文章原文
- `todolist.md` - 生成的 todolist
- `summary.md` - 核心观点总结
- `thinking.md` - AI 思考和分析
- `review.md` - 任务审计报告

### 例外情况

**只有用户明确指定时，才使用其他位置**：

| 用户指定 | 输出位置 | 示例 |
|---------|---------|------|
| "放到全局" | `~/Documents/wechat-tasks/` | 用户指定的全局目录 |
| "放到项目 X" | `项目X路径/tasks/` | 用户指定的项目目录 |
| 无指定 | `tasks/task-YYYY-MM-DD-XXX/` | 默认行为 |

### skill 文件夹独立性

**可独立复制使用**：
- 整个 `skills/wechat-article-to-todo/` 文件夹可以复制到其他项目
- 不依赖项目根目录的其他文件
- 所有路径自动适配

**复制方法**：
```bash
# 复制到其他项目
cp -r skills/wechat-article-to-todo /其他项目/.claude/skills/

# skill 会自动适配新路径
```
